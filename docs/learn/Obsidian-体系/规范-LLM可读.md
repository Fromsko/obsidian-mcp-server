# Obsidian 笔记规范（LLM/MCP 可读）

## 目标
让 LLM 在 MCP/RAG 场景下快速理解：
- 笔记存放位置
- 命名规则
- 前置元数据
- 链接关系
- 知识层级与作用

---

## 目录约定
```
learn/
  Obsidian-体系/
    README.md
    规范-LLM可读.md
    工作流-每日每周.md
    索引-总览.md
    索引-概念卡.md
    索引-问题链.md
    示例-笔记集合/
    Templates/
```

## 文件命名规则
- 中文标题 + 语义清晰
- 同一主题用统一前缀（如："索引-" "示例-"）
- 不用日期做唯一标识，除非是复盘/行动

## Frontmatter 规范
所有笔记使用 YAML 元数据，字段尽量固定：

```
---
type: 概念卡 | 读书笔记 | 问题链 | 本质拆解 | 扩展思路 | 灵感启发 | 复盘行动
status: 草稿 | 进行中 | 已完成
created: YYYY-MM-DD
source: 可选，来源链接或书名
tags: [主题标签]
---
```

## 链接规范
- 所有概念卡必须链接到：`[[索引-概念卡]]`
- 所有问题链必须链接到：`[[索引-问题链]]`
- 读书笔记必须链接至少 1 张概念卡
- 本质拆解必须链接 1 条问题链

## 知识层级
1. **读书笔记**：原始输入
2. **概念卡**：核心抽象
3. **问题链**：推理路径
4. **本质拆解**：第一性原理
5. **扩展思路**：迁移与对比
6. **灵感启发**：输出与行动

## LLM 读取建议
- 先读 `索引-总览.md` 获取全局结构
- 再读各索引页获取知识地图
- 最后深入具体笔记

---

> 本规范用于 MCP/RAG 自动检索与生成，确保语义一致与可追踪。
